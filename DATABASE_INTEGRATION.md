# Database Integration Guide for Forms

This guide explains how to connect the existing "Contact Us" and "Request Quote" forms to a database. Currently, these forms send data via an email API route (`/api/email`). This guide assumes you want to *also* store this data in a database like Supabase, PostgreSQL, or MongoDB for persistent record-keeping.

## 1. Prerequisites

Before starting, ensure you have:
1.  **A Database Instance**:
    - Recommended: **Supabase** (PostgreSQL) as it's easy to set up with Next.js.
    - Alternatives: PlanetScale, MongoDB Atlas, or a self-hosted PostgreSQL.
2.  **Environment Variables**:
    - Connection strings and API keys added to your `.env.local` file.

## 2. Supabase Setup (Recommended)

If you are using Supabase, follow these steps:

### Step 1: Create a Project
1.  Go to [supabase.com](https://supabase.com) and create a new project.
2.  Get your `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` from the project settings.

### Step 2: Create Tables
Go to the SQL Editor in Supabase and run the following to create tables for your forms:

```sql
-- Table for Contact Form Submissions
CREATE TABLE contact_submissions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  first_name TEXT,
  last_name TEXT,
  email TEXT NOT NULL,
  phone TEXT,
  country TEXT,
  interest TEXT,
  message TEXT,
  status TEXT DEFAULT 'new' -- e.g., new, contacted, closed
);

-- Table for Quote Requests
CREATE TABLE quote_requests (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  first_name TEXT,
  last_name TEXT,
  email TEXT NOT NULL,
  phone TEXT,
  country TEXT,
  trip_type TEXT,
  element_preferences JSONB, -- store destinations, travel_dates etc. as JSON or separate columns
  travelers INTEGER,
  budget TEXT,
  accommodation TEXT,
  special_requests TEXT,
  status TEXT DEFAULT 'new'
);
```

### Step 3: Install Client Library
In your project root, run:
```bash
npm install @supabase/supabase-js
```

### Step 4: Create Supabase Client
Create a file `src/lib/supabase.ts`:

```typescript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!
const supabaseKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!

export const supabase = createClient(supabaseUrl, supabaseKey)
```

## 3. Connecting the Forms

You need to modify the existing API route (`src/app/api/email/route.ts` or similar) to save data to Supabase *before* or *after* sending the email.

### Example: Modifying the API Route

Open your API route file (e.g., `src/app/api/email/route.ts`).

```typescript
import { NextResponse } from 'next/server';
import { supabase } from '@/lib/supabase'; // Import the client you created

export async function POST(request: Request) {
  const data = await request.json();
  const { firstName, lastName, email, phone, country, message, interest, ...otherData } = data;

  try {
    // 1. Insert into Database
    // Check which form is sending data based on fields present, or add a 'formType' field to your fetch body
    
    // Example for Contact Form
    if (!otherData.tripType) { 
        const { error } = await supabase
            .from('contact_submissions')
            .insert({
                first_name: firstName,
                last_name: lastName,
                email,
                phone,
                country,
                interest,
                message
            });
            
        if (error) throw error;
    } 
    // Example for Quote Request
    else {
         const { error } = await supabase
            .from('quote_requests')
            .insert({
                first_name: firstName,
                last_name: lastName,
                email,
                phone,
                country,
                trip_type: otherData.tripType,
                budget: otherData.budget,
                accommodation: otherData.accommodation,
                special_requests: otherData.specialRequests,
                // Map other fields as needed
            });

         if (error) throw error;
    }

    // 2. Send Email (Existing Logic)
    // ... keep your existing email sending code here ...

    return NextResponse.json({ message: 'Success' }, { status: 200 });

  } catch (error) {
    console.error('Error processing form:', error);
    return NextResponse.json(
      { error: 'Internal Server Error' },
      { status: 500 }
    );
  }
}
```

## 4. Security Policies (RLS)

Don't forget to enable Row Level Security (RLS) on your Supabase tables to prevent unauthorized access.
- **Enable RLS** on both tables.
- **Create Policy**: "Enable INSERT for anon users" (so anyone can submit a form).
- **Create Policy**: "Enable SELECT for authenticated users only" (so only admin dashboard users can read submissions).
